isPSkip judges whether all processes of an annotated program are ASkip, i.e. P ?= (ASkip || ... || ASkip). If a program satisfies the isPSkip property, then it is regarded to be termination.
Definition isPSkip (pe: Progexe) :Prop :=
  ∀ i ac, PEMod.get pe i = Some ac → ac = ASkip.

TSim ((C, σp) ≼ (C', σp')) is the simulation relation between two pairs of an annotated command and a private state.
Similar with the stardard simulation, two cases are considered here:
  • If C can execute one step forward, then C' can execute one or more steps and these two remaining commands satisfy the simulation relation.
  • If C is Skip, then C' can execute to Skip.
Note that for each corresponding executions of two commands, the initial shared states and the resulting shared states are required to be equivalent. And the terminated private states are required to be equivalent. seqstateII is the invariant during the execution. it states that if one value is influenced by process i, then the send permission of the current process must be less than that of process i.
We assume that two simulated commands must terminate.

CoInductive TSim (ac ac': AnnoCmd) (pst pst': PrivateState) (t: Tid) (l: Label)
            (pc: Label) (envp': Env) (rho: Policys) (envs: Env) (i: Tid) :Prop :=
| TSimProceed:
    (
      (
        ∀ sst sst' ac1 pst1 sst1,
          SharedStateApprox sst sst' i l rho envs →
          SeqStep t rho (ac, (pst, sst)) (ac1, (pst1, sst1)) →
          
          
          (seqstateII (pst, sst) t rho) →
          (seqstateII (pst', sst') t rho) →
          ∃ ac1' pst1' sst1',
            starSeqStep t rho (ac', (pst', sst')) (ac1', (pst1', sst1')) ∧
            SharedStateApprox sst1 sst1' i l rho envs ∧
            TSim ac1 ac1' pst1 pst1' t l pc envp' rho envs i ∧
            seqstateII (pst1, sst1) t rho ∧
            seqstateII (pst1', sst1') t rho
      )
      ∧
      (
        ∀ sst sst',
          ac = ASkip →
          SharedStateApprox sst sst' i l rho envs →
          
          (seqstateII (pst', sst') t rho) →
          ∃ pst1' sst1',
            starSeqStep t rho (ac', (pst', sst')) (ASkip, (pst1', sst1')) ∧
            SharedStateApprox sst sst1' i l rho envs ∧
            PStateApprox pst pst1' l pc envp' ∧
            seqstateII (pst1', sst1') t rho
      )
    )
    →
    TSim ac ac' pst pst' t l pc envp' rho envs i.

PSim((P, Σp) ≼ (P', Σp')) is the simulation relation between two pairs of an annotated program and a private state mapping. Its definition is similar with TSim.

CoInductive PSim (pe pe': Progexe) (psts psts': PrivateStateSet) (l: Label)
            (pcs: PC_Map) (envps': Env_Map) (rho: Policys) (envs: Env) (i: Tid) :Prop :=
| PSimProceed:
    (
      (
        ∀ sst sst' pe1 psts1 sst1,
          SharedStateApprox sst sst' i l rho envs →
          ParaStep rho (pe, (psts, sst)) (pe1, (psts1, sst1)) →
          
          
          stateII (psts, sst) rho →
          stateII (psts', sst') rho →
          ∃ pe1' psts1' sst1',
            starParaStep rho (pe', (psts', sst')) (pe1', (psts1', sst1')) ∧
            SharedStateApprox sst1 sst1' i l rho envs ∧
            PSim pe1 pe1' psts1 psts1' l pcs envps' rho envs i ∧
            stateII (psts1, sst1) rho ∧
            stateII (psts1', sst1') rho
      )
      ∧
      (
        ∀ sst sst',
          isPSkip pe →
          SharedStateApprox sst sst' i l rho envs →
          
          stateII (psts', sst') rho →
          ∃ psts1' sst1',
            starParaStep rho (pe', (psts', sst')) (peSkip pe', (psts1', sst1')) ∧
            SharedStateApprox sst sst1' i l rho envs ∧
            PStatesApprox psts psts1' l pcs envps' ∧
            stateII (psts1', sst1') rho
                    
      )
    )
    →
    PSim pe pe' psts psts' l pcs envps' rho envs i.
