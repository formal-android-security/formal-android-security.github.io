Require Import Lang.

Unlike starSeqStep which relates two pairs of a command and a state, including a private state and a shared state, starparaSeqStep relates two pairs of a command and a private state. During the execution, the shared state may be changed arbitrarily. starSeqStep can be regarded as a special case of starparaSeqStep.
This relation satisfies a property called kparaSeqStep_Skip_tpc_eq which is crucial to prove the simulation of if and while commands. Informally, this property states that
if C is well typed and (C, (tpc, ps)) ↪* (ASkip, (tpc', ps')), then tpc = tpc'.

Fixpoint kparaSeqStep (k:nat) (t: Tid) (rho: Policys) (sc: AnnoCmd×PrivateState) (sc':AnnoCmd×PrivateState): Prop :=
  match k with
  | S n ⇒ match (sc, sc') with
           | ((ac,pst), (ac',pst')) ⇒
             ∃ sst ac'' pst'' sst'',SeqStep t rho (ac, (pst,sst)) (ac'', (pst'',sst''))
                                         ∧ kparaSeqStep n t rho (ac'',pst'') (ac',pst')
           end
    | O ⇒ sc = sc'
  end.

Function starparaSeqStep (t: Tid) (rho: Policys) (psc:(AnnoCmd×PrivateState)) (psc':AnnoCmd×PrivateState) : Prop :=
  ∃ k, kparaSeqStep k t rho psc psc'.
